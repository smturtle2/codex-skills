# Prompt Patterns

Use these patterns as source material for the prompts generated by `prepare_animation_run.py`. Keep final `$image-creator` prompts concise and English.

## Base Character Prompt

```text
Create one clean full-body reference image for an animation character named <name>.

Character: <character description>.
Style: <style notes>.

Output a single centered full-body character pose on a perfectly flat <chroma name> <chroma hex> chroma-key background. The character must be fully visible, readable, and suitable for generating animation frames. Do not include scenery, text, labels, borders, ground planes, floors, cast shadows, contact shadows, oval floor shadows, landing marks, glows, detached effects, or extra props not requested. Do not use the chroma-key color or colors close to it in the character, outfit, props, highlights, shadows, or effects.
```

## Action Grid Prompt

```text
Create one animation frame grid for the character in the action `<action-id>`.

Use the attached canonical base image as the character identity source. Use the attached layout guide only for frame count, grid cell spacing, centering, and safe padding. Do not copy guide lines, boxes, labels, colors, or the guide background into the output.

Identity lock:
- Do not redesign the character.
- Preserve the same head shape, face, proportions, silhouette, markings, palette, outfit, materials, and props from the canonical base.
- Change only pose, expression, and action timing needed for `<action description>`.

Output exactly <frame-count> separate full-body animation frames arranged in a <columns>x<rows> grid, read left-to-right and top-to-bottom.

Layout requirements:
- Treat the image as <columns>x<rows> equal-size invisible grid cells.
- Fill the first <frame-count> cells with exactly one complete centered pose each.
- Keep every body part inside the cell's safe area.
- No pose may cross into a neighboring cell.
- Leave unused cells empty chroma-key if the layout has more cells than frames.
- Use a perfectly flat <chroma name> <chroma hex> chroma-key background.
- Do not include any ground plane, floor line, cast shadow, contact shadow, oval floor shadow, landing mark, dust, glow, speed line, motion trail, smear, detached symbol, loose effect, visible grid, box, frame number, text, watermark, or scenery.
- Show weight, jumping, or landing only through the character pose. Do not draw floor cues.
- Keep the first and last frames compatible for a clean loop when loop mode is enabled.
```

## Repair Prompt Addendum

Append this to an action prompt when regenerating a failed action:

```text
Repair note:
- The previous grid failed because: <reason>.
- Regenerate the entire action grid, not a single frame.
- Keep the same canonical character identity.
- Fix clipping, empty cells, cell crossing, detached fragments, guide artifacts, and uneven spacing.
- Remove all ground planes, floor cues, cast shadows, contact shadows, oval floor shadows, landing marks, and dust.
- Preserve the action timing and requested frame count.
```
